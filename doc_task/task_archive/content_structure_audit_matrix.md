# 魚油內容架構：多維度極限壓力測試 (Multi-Perspective Audit)

這份文件從 5 個互相衝突的視角，對目前的 [內容架構](task_new-post_content_structure.md) 進行「紅隊測試」，以確保沒有盲點。

---

## 🎭 角色 1：Google 演算法 (The SEO Bot)
> **關注點**：爬蟲路徑、語義理解、關鍵字蠶食、結構化數據

| 檢查項目 | 評估結果 | 優化建議 |
| :--- | :--- | :--- |
| **關鍵字蠶食 (Cannibalization)** | ✅ **Pass**：各文章主關鍵字已區隔 (功效/挑選/推薦/價格/送禮)，無重疊。 | 需確保 `Title Tag` 寫法差異化，例如支柱文強調「全指南」，評測文強調「排行榜」。 |
| **語義關聯 (NLP)** | ✅ **Pass**：已包含大量共現詞 (LSI)，如「發炎」、「三酸甘油酯」、「rTG」。 | 可在支柱文章增加 `FAQ Schema`，直接佔領 Google 搜尋結果頁的版位。 |
| **連結深度 (Crawl Depth)** | ⚠️ **Risk**：目前「行為效益」與「送禮」較為孤立。 | 需在「支柱文章」與「評測文章」的側邊欄或文末，強制加入這兩篇的情境導流。 |
| **搜尋意圖 (Intent)** | ✅ **Pass**：精準對應 Informational (支柱), Commercial (評測), Transactional (SOP)。 | - |

## 🎭 角色 2：多疑的消費者 (The Skeptic User)
> **關注點**：信任感、真實性、負面資訊、是不是詐騙

| 檢查項目 | 評估結果 | 優化建議 |
| :--- | :--- | :--- |
| **利益揭露 (Transparency)** | ⚠️ **Risk**：評測文若只說好話，會被認為是業配。 | **必須**在評測文中加入「缺點揭露」區塊，例如某款雖純度高但「很大顆」、某款雖便宜但「有腥味」。 |
| **引經據典 (Citation)** | ✅ **Pass**：支柱文章強調 2026 科學實證。 | 建議將參考文獻 (PubMed/IFOS) 的連結設為 `nofollow` 但保留，增加學術權威感。 |
| **負面預期管理** | ✅ **Pass**：已加入「腥味」、「難吞」的感官測試。 | 這是建立信任的最強武器，務必拍出真實照片 (如剪開膠囊)。 |
| **過度承諾 (Over-promise)** | ✅ **Pass**：已修正「被動收入」等誇大用語。 | 保持「輔助」基調，避免使用「治療」、「痊癒」等字眼。 |

## 🎭 角色 3：商業轉換專家 (The CRO Specialist)
> **關注點**：流量變現、客單價 (AOV)、漏斗流向

| 檢查項目 | 評估結果 | 優化建議 |
| :--- | :--- | :--- |
| **行動呼籲 (CTA)** | ⚠️ **Risk**：支柱文章雖然權威，但讀者看完可能就走了。 | 在支柱文章的每個 H2 結尾加入「軟性 CTA」，如：「想知道哪款魚油符合這項標準？看評測➜」。 |
| **組合銷售 (Upsell)** | ✅ **Pass**：投資組合 (Portfolio) 策略非常完美。 | 建議在「評測文」的推薦清單中，直接列出「魚油+B群」的懶人組合包連結。 |
| **跳出率 (Bounce Rate)** | ✅ **Pass**：SOP 文嵌入產品卡片的策略是關鍵。 | 確保這些嵌入卡片有明顯的「查價」按鈕，引導進入電商頁面。 |

## 🎭 角色 4：UX 設計師 (The Designer)
> **關注點**：易讀性、手機體驗、視覺化呈現

| 檢查項目 | 評估結果 | 優化建議 |
| :--- | :--- | :--- |
| **資訊密度 (Scannability)** | ⚠️ **Risk**：SOP 與評測文充滿數據，手機閱讀易疲勞。 | 所有比較表 (Table) 必須支援 **RWD (響應式)**，手機版建議改為「卡片滑動」而非表格。 |
| **視覺錨點 (Visual Anchors)** | ⚠️ **Risk**：目前文字偏多。 | 強烈建議製作「魚油挑選流程圖 (Flowchart)」放在 SOP 文開頭，讓讀者 3 秒看懂。 |
| **導航體驗 (Navigation)** | ✅ **Pass**：目錄結構清晰。 | 文章開頭需加入「閱讀進度條」或「懸浮目錄」，方便讀者隨時跳轉。 |

## 🎭 角色 5：醫療審稿人 (The Medical Editor)
> **關注點**：法規合規 (YMYL)、正確性、劑量安全

| 檢查項目 | 評估結果 | 優化建議 |
| :--- | :--- | :--- |
| **紅線字眼 (Red Flags)** | ✅ **Pass**：已加入「抗凝血劑」警語。 | 需再次全站掃描，確保沒有出現「抗癌」、「降血壓 (需改為調節)」等醫療效能詞。 |
| **特殊族群 (Vulnerable)** | ✅ **Pass**：已涵蓋孕婦、兒童。 | 針對「孕婦」段落，建議加入「產檢醫師諮詢」的免責聲明，增加安全係數。 |
| **劑量建議 (Dosage)** | ✅ **Pass**：已區分保健與醫療劑量。 | 針對「高劑量」使用者，務必提醒每日上限 (台灣法規 2000mg/日)。 |

## 🏆 總結與最終優化指令 (Action Items)

基於以上 5 種視角的審查，我們需要補強最後 3 個短板：

1.  **UX 優化**：製作「RWD 比較表」與「挑選流程圖」的視覺規範。
2.  **轉換優化**：在支柱文章中埋設「軟性 CTA」導流至評測文。
3.  **信任優化**：在評測文中強制加入「缺點揭露 (Cons)」欄位。
