## 3. 行銷與體驗優化 (Experience & Conversion)

### 3.1 智慧選購路徑 (Buying Guide Workflow)
在 Hub Page 置入 **「3 題選購測驗」** (Interactive Quiz)，取代被動連結：
1.  **需求定位**: (護眼 / 心血管 / 懷孕 / 情緒)
2.  **痛點篩選**: (是否害怕吞大膠囊？)
3.  **飲食習慣**: (每週吃魚頻率？)
-> **Result**: 直接輸出「你是高濃度小顆粒派」，並推薦對應文章或產品。

### 3.2 強化「人味」實測 (The Lab Test)
打破冷冰冰的醫學形象，加入編輯部真實體驗。
- **溶解/乳化實驗**: 拍攝影片/GIF，視覺化呈現吸收潛力。
- **顆粒尺規圖**: 直接拿尺量，放在硬幣旁對比 (解決 "吞嚥困難" 痛點)。
- **腥味盲測**: 真實評分 (1-5分)，不只看數據，更看感受。

### 4.2 Core: 基礎保養指南 (The Knowledge Base)
**檔案路徑**: `post/fish-oil.html`
**定位**: 建立知識權威。解釋「為什麼」，並遵循 `doc/writing-guide.md` 之標準文章結構。

#### B1. H1 Strategy (權威標題策略)
- **結構**: [核心關鍵字] [強烈動詞]：[功效1]、[功效2]... [目標族群/終極指南]
- **範例**: 魚油功效總整理：不只護眼與降血脂！還能改善憂鬱、抗發炎與調節血壓的完整解析

#### B2. Core Modules (核心模組深度編排)
必須包含以下標準模組，並搭配指定 UI 元件：

- **Module 1: 定義與化學結構**
  - **標題**: `<h2> 魚油是什麼？Omega-3 的黃金標準`
  - **內容**: 解釋 EPA/DHA 的定義，與 ALA (亞麻仁油) 的轉換率與生物利用率比較。
  
- **Module 2: 核心功效總覽 (The Benefits)**
  - **標題**: `<h2> 魚油對人體有哪些好處？`
  - **UI 元件**: `<div class="info-cards">` (需列出 5-8 點)
  - **內容**: 涵蓋 心血管 (JELIS研究)、眼睛 (乾眼症/視網膜)、大腦 (記憶/情緒)、抗發炎機制等。需引用 PubMed/JAMA 數據。
  
- **Module 3: 攝取量與時間 (Timing & Dosage)**
  - **標題**: `<h2> 每日建議攝取量與最佳時間`
  - **內容**: 引用 WHO/FDA/台灣衛福部 DRIs 建議量。強調「隨餐吃」的脂溶性特性。
    
- **Module 4: 食物來源 (Food Sources)**
  - **標題**: `<h2> 哪些天然食物富含 Omega-3？`
  - **UI 元件**: `<div class="data-table">` (需包裹 `.responsive-table-wrapper`)
  - **內容**: 比較 深海魚 (鮭魚/鯖魚/秋刀魚) vs 植物來源 (亞麻籽/核桃) 的含量差異。
  
- **Module 5: 缺乏警訊與高風險族群 (The Need)**
  - **標題**: `<h2> 誰最需要補充？Omega-3 缺乏的警訊`
  - **UI 元件**: `<div class="quick-test">` (自我檢測) 或 `<div class="risk-group-cards">`
  - **內容**: 外食族 (Omega-6 失衡)、懷孕婦女、老年人、乾眼症狀者。
  
- **Module 6: 挑選指南 (Selection Guide)**
  - **標題**: `<h2> 如何挑選魚油推薦產品？`
  - **UI 元件**: `<div class="comparison-table">` 或 Checklist
  - **內容**: 
    - 濃度 (Concentration): 80% 以上是門檻。
    - 型態 (Form): rTG vs EE vs TG 的吸收率差異。
    - 檢驗 (Quality): IFOS 五星、新鮮度 (PV/AV值)。
    
- **Module 7: 風險與禁忌 (Risks)**
  - **標題**: `<h2> 魚油的副作用與藥物交互作用`
  - **內容**: 抗凝血劑 (Warfarin) 交互作用警示、近期手術者停用建議。
  
- **Module 8: 常見問題 (FAQ)**
  - **標題**: `<h2> 常見問題`
  - **技術**: 注入 JSON-LD FAQPage Schema 以搶佔 PAA 版位。
  - **內容**: 藻油跟魚油差別？過期能吃嗎？跟鈣片可以一起吃嗎？

### 4.4 Technical Deep Dives: 技術解析 (The Science)

#### D1. 型態結構解析 (`post/doing/fish-oil-structure.html`)
- **定位**: rTG vs EE vs TG 的終極比較。
- **視覺化圖解**: 生化結構圖 (三腳架 vs 乙基)，解釋吸收率差異。
- **CP值分析**: EE 雖然便宜但吸收率低，換算真實成本是否划算？

#### D2. 品質與檢驗 (`post/doing/fish-oil-quality.html`)
- **定位**: 讓消費者學會看檢驗報告。
- **IFOS 認證**: 解釋五星標準是什麼 (重金屬、氧化指標)。
- **濃度計算**: 破解「每顆膠囊重」vs「Omega-3 淨重」的陷阱。
- **新鮮度**: 過氧化價 (PV) 與 茴香胺價 (AV) 的意義。

#### D3. 來源比較 (`post/doing/fish-oil-sources.html`)
- **定位**: 魚油 vs 藻油 vs 磷蝦油。
- **素食方案**: 什麼是裂壺藻？吸收率跟魚油一樣嗎？
- **磷蝦油**: 磷脂質結構的優勢與缺點 (濃度通常較低)。

## 5. 工程開發指南 (Engineering & Schema Implementation)

本區塊專供工程開發人員參考，包含結構化數據建置與前端效能規範。

### 5.1 結構化數據 (JSON-LD Schema)
資深工程師需介入 HTML 頭部 (`<head>`)，注入 JSON-LD 以搶佔 Google 版位。

- **FAQ Schema**: 針對 `fish-oil.html` 的常見問答 (飯前/飯後吃？)。
- **MedicalWebPage Schema**: 針對所有核心文章，標註 `reviewedBy` (醫師/藥師)，強化 E-E-A-T。
- **Comparison Table Schema**: 針對 `fish-oil-market-analysis.html` 的產品評測表。

#### FAQ Schema 代碼範例
```json
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "name": "rTG 型態的魚油吸收率真的比較好嗎？",
    "acceptedAnswer": { "text": "是的，研究顯示 rTG 型態的吸收率是 EE 型態的 2 倍以上..." }
  }]
}
</script>
```

### 5.2 前端效能標準 (Frontend Performance)
- **Core Web Vitals**: 工具頁面 (Tools) 需採用 **Progressive Enhancement**。靜態優先，JS 延遲載入 (IntersectionObserver)，確保 LCP 不受影響。
- **Anti-Fragile**: 計算機需加入 Input Validation (如：防止輸入 200 顆) 與 Debounce 機制。

### 5.3 自動化內部連結 (Internal Linking)
- **Glossary Automation**: 建立 `glossary.json`，文章中出現 "rTG", "IFOS" 等關鍵字時，自動生成 Tooltip 與內部連結，增加網站深度。
