## 3. 行銷與體驗優化 (Experience & Conversion)

### 3.1 智慧選購路徑 (Buying Guide Workflow)
在 Hub Page 置入 **「3 題選購測驗」** (Interactive Quiz)，取代被動連結：
1.  **需求定位**: (護眼 / 心血管 / 懷孕 / 情緒)
2.  **痛點篩選**: (是否害怕吞大膠囊？)
3.  **飲食習慣**: (每週吃魚頻率？)
-> **Result**: 直接輸出「你是高濃度小顆粒派」，並推薦對應文章或產品。

### 3.2 強化「人味」實測 (The Lab Test)
打破冷冰冰的醫學形象，加入編輯部真實體驗。
- **溶解/乳化實驗**: 拍攝影片/GIF，視覺化呈現吸收潛力。
- **顆粒尺規圖**: 直接拿尺量，放在硬幣旁對比 (解決 "吞嚥困難" 痛點)。
- **腥味盲測**: 真實評分 (1-5分)，不只看數據，更看感受。

### 4.3 Verticals: 分眾與專業指南 (Deep Dives)

#### C1. 分眾總覽 (`post/doing/fish-oil-groups.html`)
- **定位**: 針對不同生命階段 (Life Stages) 的劑量與選購建議。
- **孕婦區塊**: 藻油 vs 魚油的重金屬權衡、EPA/DHA 黃金比例 (避免早產)。
- **兒童區塊**: 顆粒大小、口味接受度 (去除腥味技術)、智力發展數據。
- **學生/上班族**: 腦力與視力保養重點。

#### C2. 銀髮心血管專區 (`post/doing/fish-oil-senior.html`)
- **定位**: 心血管保養與藥物交互作用。
- **警示卡 (Red Alert)**: 紅麴/抗凝血劑 (Warfarin) 搭配禁忌，強調諮詢醫師。
- **高濃度 EPA**: 引用 JELIS 研究，解釋為何需要純 EPA。
- **吞嚥友善**: 推薦小顆粒 (Mini Gels) 或液態魚油。

#### C3. 常見問答 (`post/doing/fish-oil-faq.html`)
- **定位**: 快速解答零碎問題，攔截長尾關鍵字 (Long-tail Keywords)。
- **分類FAQ**: 依照 "食用時機", "副作用", "保存方式" 分類。
- **Schema**: 必須包含完整的 JSON-LD FAQPage 標記。

#### C4. 食用指南 (`post/doing/fish-oil-guide.html`)
- **定位**: 使用手冊 (How-to)。
- **最佳時機**: 隨餐吃的科學原理 (脂溶性)。
- **副作用排除**: 發生打嗝魚腥味怎麼辦？拉肚子怎麼辦？
- **保存方式**: 冰箱冷藏 vs 陰涼處？膠囊霧化還能吃嗎？

### 4.4 Technical Deep Dives: 技術解析 (The Science)

#### D1. 型態結構解析 (`post/doing/fish-oil-structure.html`)
- **定位**: rTG vs EE vs TG 的終極比較。
- **視覺化圖解**: 生化結構圖 (三腳架 vs 乙基)，解釋吸收率差異。
- **CP值分析**: EE 雖然便宜但吸收率低，換算真實成本是否划算？

#### D2. 品質與檢驗 (`post/doing/fish-oil-quality.html`)
- **定位**: 讓消費者學會看檢驗報告。
- **IFOS 認證**: 解釋五星標準是什麼 (重金屬、氧化指標)。
- **濃度計算**: 破解「每顆膠囊重」vs「Omega-3 淨重」的陷阱。
- **新鮮度**: 過氧化價 (PV) 與 茴香胺價 (AV) 的意義。

#### D3. 來源比較 (`post/doing/fish-oil-sources.html`)
- **定位**: 魚油 vs 藻油 vs 磷蝦油。
- **素食方案**: 什麼是裂壺藻？吸收率跟魚油一樣嗎？
- **磷蝦油**: 磷脂質結構的優勢與缺點 (濃度通常較低)。

### 4.5 Utility & Review: 工具與評測 (Action & Proof)

#### E1. 挑選工具箱 (`post/doing/fish-oil-tools.html`)
- **定位**: 讓數據說話，解決「怎麼算」的問題。
- **真實成本計算機**: (價格 ÷ Omega3濃度) = 真實CP值。

- **結果保存**: 生成 "我的處方籤" 圖片分享。

#### E2. 市場評測 (`post/doing/fish-oil-market-analysis.html`)

- **編輯部實測 (Lab Test)**: 剪開膠囊影片、腥味盲測雷達圖、顆粒大小尺規圖。
- **一鍵比較表**: 10款熱門產品橫向對比 (濃度/型態/單日成本)。

## 5. 工程開發指南 (Engineering & Schema Implementation)

本區塊專供工程開發人員參考，包含結構化數據建置與前端效能規範。

### 5.1 結構化數據 (JSON-LD Schema)
資深工程師需介入 HTML 頭部 (`<head>`)，注入 JSON-LD 以搶佔 Google 版位。

- **FAQ Schema**: 針對 `fish-oil.html` 的常見問答 (飯前/飯後吃？)。
- **MedicalWebPage Schema**: 針對所有核心文章，標註 `reviewedBy` (醫師/藥師)，強化 E-E-A-T。
- **Comparison Table Schema**: 針對 `fish-oil-market-analysis.html` 的產品評測表。

#### FAQ Schema 代碼範例
```json
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "name": "rTG 型態的魚油吸收率真的比較好嗎？",
    "acceptedAnswer": { "text": "是的，研究顯示 rTG 型態的吸收率是 EE 型態的 2 倍以上..." }
  }]
}
</script>
```

### 5.2 前端效能標準 (Frontend Performance)
- **Core Web Vitals**: 工具頁面 (Tools) 需採用 **Progressive Enhancement**。靜態優先，JS 延遲載入 (IntersectionObserver)，確保 LCP 不受影響。
- **Anti-Fragile**: 計算機需加入 Input Validation (如：防止輸入 200 顆) 與 Debounce 機制。

### 5.3 自動化內部連結 (Internal Linking)
- **Glossary Automation**: 建立 `glossary.json`，文章中出現 "rTG", "IFOS" 等關鍵字時，自動生成 Tooltip 與內部連結，增加網站深度。
