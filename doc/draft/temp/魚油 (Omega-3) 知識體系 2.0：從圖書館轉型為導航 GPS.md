# **魚油 (Omega-3) 知識體系深度架構：專業優化建議**

## **1\. SEO 戰略升級 (Topical Authority 2.0)**

### **關鍵字與搜尋意圖對齊**

目前的 Level 2 與 Level 3 應該進一步細化以涵蓋完整的搜尋漏斗：

* **資訊型意圖 (Informational):** 「魚油功效」、「EPA vs DHA 分別」。  
* **調查型意圖 (Commercial Investigation):** 「魚油推薦 2025」、「rTG 魚油比較」。  
* **交易型意圖 (Transactional):** 「高純度魚油購買」、「小顆粒魚油推薦」。

### **結構化數據 (Schema Markup) 的極致運用**

資深工程師視角：不僅是寫內容，還要在 HTML 中注入 **JSON-LD**。

* **FAQ Schema:** 針對 fish-oil.html 的常見問答。  
* **MedicalWebPage Schema:** 標註審核醫生、醫學專長。  
* **Comparison Table Schema:** 針對 fish-oil-structure.html 中的規格對比。

## **2\. 行銷與轉換優化 (Conversion Rate Optimization)**

### **從「決策支持」進化到「選購路徑」**

* 新增：智慧選購路徑 (Buying Guide Workflow)  
  在 Hub 頁面不只是放連結，建議加入一個「3 題找出適合你的魚油」的互動元件。  
  * Q1: 你的主要需求是？（心血管 / 眼睛 / 懷孕 / 情緒）  
  * Q2: 你是否害怕吞大膠囊？  
  * Q3: 你的日常飲食是否常吃魚？  
  * **輸出:** 直接導向對應的精準方案頁面。

### **強化第一層「人味」 (EEAT 中的 Experience)**

* **實測數據:** 除了 PubMed 數據，加入「編輯部 30 天實測」或「100 位受試者口感回饋」。醫學專業讓人信任，但真實體驗讓人買單。

## **3\. 工程實作優化 (Performance & Engineering)**

### **核心網頁指標 (Core Web Vitals) 監控**

* **動態儀表板優化:** 數據儀表板應採用 **Progressive Enhancement (漸進增強)**。首屏先顯示靜態圖表，當使用者滾動到該區域時才加載 JS 計算逻辑，確保 LCP 評分。  
* **計算機技術選型:** 使用純 Vanilla JS 或輕量級框架，避免引入大型 Library。計算邏輯應包含 debounce 與輸入驗證，防止計算錯誤導致的信任危機。

### **知識圖譜 (Knowledge Graph) 的技術展現**

* 建議在 A-Z Glossary 頁面實作 **內部鏈接自動化**。當文章中出現 rTG 或 IFOS 時，自動生成 Tooltip 或輕量連結，增強 Site Depth。
